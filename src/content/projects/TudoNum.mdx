## Project Overview

**TudoNum** (live at tudonum.app) is a multi-service super app architected from the ground up to span professional services, ride-hailing, logistics, and deliveries. It's built with a Python/Django REST backend and React + Vite frontend maintained as independent codebases, enabling teams to work and scale independently while sharing a unified data model.

**My role**: Senior Full Stack Developer, architecting and shipping the entire end-to-end professional services module including vendor onboarding, RBAC, booking engine, and payments infrastructure.

### Core Purpose

TudoNum eliminates the need for multiple apps by providing access to services across a unified ecosystem:

- **Professional Services** - Consulting, legal services, freelancing, vendor onboarding
- **Transportation** - Taxi, ride-hailing, logistics
- **Food & Groceries** - Food delivery, grocery shopping
- **Healthcare** - Doctor consultations, health services
- **Real Estate** - Property buying, selling, renting
- **Beauty & Wellness** - Salons, spas, personal care
- **Education** - Tutoring, online courses
- **Entertainment** - Gaming, entertainment services
- **Marketplace** - Classified ads, e-commerce
- **Home Services** - Maintenance, construction

### Key Architectural Achievements

#### **Schema-Driven Form Engine ("DNA")**

The core innovation is a dynamic form engine where a single JSON configuration drives React to render complex, multi-step forms from a shared UI primitive library. This enables:

- Hundreds of distinct vendor and service flows without new component code
- Consistent UX across the platform
- Rapid iteration on new service types
- Reuse across Company Onboarding pipeline (document collection, KYC checks, admin approval state machines)

#### **Professional Services Module (My Ownership)**

End-to-end implementation including:
- Vendor onboarding and profile management
- Multi-tier RBAC with in-app role switching (customer / vendor / admin)
- City- and country-scoped service discovery
- Calendar-driven booking with survey scheduling
- Real-time availability and conflict detection across provider calendars
- Wallet-backed checkout with secure transactions

#### **Multi-Region Payment Rails**

Integrated payment infrastructure supporting:
- Stripe for international cards
- WalPay for regional coverage
- Local gateways for Oman, Portugal, and Pakistan
- Internal wallet ledger handling top-ups, transfers, and provider payouts

#### **AI-Powered Admin Panel**

Built on Claude Code, OpenAI, and OpenRouter SDKs:
- Automated triage and support routing
- Content generation and optimization
- Operational tooling and insights
- Transactional SMTP and webhook handlers for invitations, verification, and payment events

#### **Production Infrastructure**

End-to-end ownership:
- Frontend deployed on Vercel with ISR caching
- Django backend on AWS with auto-scaling
- CI/CD pipelines with environment management
- Zero-downtime release workflows
- Comprehensive monitoring and error tracking
